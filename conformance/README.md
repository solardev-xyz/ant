# Conformance vectors

Cross-implementation test vectors for ant, generated by calling the
canonical bee implementation directly.

- [`vectorgen/`](vectorgen/) — Go generator. It depends on the sibling
  `bee` checkout via a `replace` directive (`../../../bee`), so every
  derived value (CAC addresses, SOC signatures, feed ids, stamp
  signatures, parity tables) comes from bee's own code, not a
  reimplementation. The only mirrored details are the byte layouts of
  bee's *private* feed-index marshallers, cited to source lines in
  `main.go`.
- [`vectors/`](vectors/) — generated JSON, checked in so the Rust tests
  run without a Go toolchain or the bee checkout.

The consuming tests live in
[`crates/ant-conformance`](../crates/ant-conformance) and assert ant is
bit-exact against the vectors (`cargo test -p ant-conformance`).

Regenerate after bumping the bee checkout:

```sh
cd conformance/vectorgen
go run . -out ../vectors
cargo test -p ant-conformance
```

Known doc-vs-code drift captured here: bee's PARANOID redundancy level
is 89 parities / **39** max data shards per 128-slot node
(`GetParities(128) == 89`), while docs.ethswarm.org describes it as
90/38. The vectors follow the implementation.
